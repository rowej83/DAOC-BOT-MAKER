- REQUIRED STEPS IN-GAME -

Step 1: The bot assumes you are using the default, classic theme. Not BobsUI or any other custom theme.

Step 2: The game MUST be in windowed mode with a resolution smaller then the desktop's.

Step 3: The bot does not read memory addresses, instead, it uses exact pixel locations to generate it's data and to use in it's decision making process. Because of this, a program named Gridy is required. As described by it's creators: Gridy adds an invisible grid to Windows, so that any window you move or resize is snapped to it. Snapping is enabled with mouse drag or keyboard arrow keys. What this allows us to do is to be sure that once our windowed DAOC screen is created, we can position it back to an exact location so the pixel coordinates do not have to be updated everytime you start or play the game. I will post a video tutorial on this to explain better.

- CUSTOM MAKE YOUR BOT -

Step 1: This tool is used to custom make a macro that requires AC-Tools to run.

Step 2: Enter your keyboard configuration under the Keyboard Config Setup section so the bot knows which buttons to press.

Step 3: Enter your color coordinates so that the bot can be aware of your character's state (i.e. Engaged, if it has a target, full health etc..). This section will soon have a video tutorial on how to find the required coordinates.

Step 4: Once all the required fields are filled out you may view the generated AC Tools Bot or have it copied to your clipboard. At which point you can open AC Tools and paste it into a new macro to be saved. If the configuration is properly set, there should be no further actions necessary.

- TIPS WHILE USING THE BOT -

Step 1: Since the bot does not include movement of your character, you will need a camp of mobs that are within targeting reach by use of TAB. Also keep in mind to be far enough to not get aggro. However, the bot will react to aggro while your healing and will kill the mob but you should avoid aggro in the first place.

Step 2: The bot should not be left unattended incase people attempt to communicate you as this will be the only way to know if your actual playing or not.

Step 3: Once all the required fields are filled out you may view the generated AC Tools Bot or have it copied to your clipboard. At which point you can open AC Tools and paste it into a new macro to be saved. If the configuration is properly set, there should be no further actions necessary.

Constants
  pullkey= {{bot.pullKey}} //enter pullkey i.e. hotbar 1 would be 1
  switch={{bot.switchKey}}
  attackkey= {{bot.attackKey}} //enter attack key
  specialattackkey= {{bot.bludKey}} //enter special attack key
  specialattackkey2= {{bot.contKey}}
  restkey= {{bot.restKey}} //your sit key
  tabkey={TAB}
  facekey={{bot.faceKey}}
  sprintkey={{bot.sprintKey}}
  // do no edit below
  fullhp=1
  havetarget=0
  isengaged=0
  isstanding=1
  RandomInteger = 0
  Maxval = 0
  Minval = 0
  minmp=1
  mobhp=1
  endo=1
  //leave the 7 above vars alone
End
Procedure setactive
  SetActiveWindow Dark Age of Camelot © 2001-2011 Electronic Arts Inc. All Rights Reserved.
End
Procedure removepopup
  IsWindow Sponsored session
    // Do these commands if true
    SetActiveWindow Sponsored session
    delay 1000
    keys {return}
    delay 1000
    SetActiveWindow Dark Age of Camelot © 2001-2011 Electronic Arts Inc. All Rights Reserved.
  Else
  End
end
Procedure hastarget
  loadrgb {{bot.targetx}}, {{bot.targety}}
  if {rgbred} < 20
    setconst havetarget=0
  else
    setconst havetarget=1
  end //end of if
End //end of hastarget
Procedure checkengaged
  loadrgb {{bot.engagedx}}, {{bot.engagedy}}
  if {rgbblue} > 10
    setconst isengaged=0
  else
    setconst isengaged=1
  end
End //end checkengaged
Procedure checkfullhp
  loadrgb {{bot.fullhpx}}, {{bot.fullhpy}}
  if {rgbred} <200
    setconst fullhp=0
  else
    setconst fullhp=1
  end
End// end of checkfullhp
Procedure checkmobhp
  loadrgb {{bot.mobhpx}}, {{bot.mobhpy}}
  if {rgbred} <100
    setconst mobhp=0 //if mob hp less then or 30%
  else
    setconst mobhp=1 //if mob hp is full then it is 1
  end
End// end of checkfullhp
Procedure checkendo
  loadrgb {{bot.minendox}}, {{bot.minendoy}}
  if {rgbgreen} >100
    setconst endo=1 //has enough endo
  else
    setconst endo=0 //if mob hp is full then it is 1
  end
end
Delay 2000
While 1=1
  Call setactive
  call removepopup
  Call checkHP
  call removepopup
  Call setactive
  Call Pull
  call removepopup
  Call setactive
  Call Fight
  call removepopup
  Call setactive
End// end of main while loop
Procedure checkHP
  Call checkfullhp
  Call checkendo
  if $fullhp=1 and $endo=1 //has full hp, do nothing
    if $isstanding=0
      keydown $restkey 500
      SetConst ISSTANDING = 1
      Delay 1000
      exit
    else //else of isstandin=0
      SetConst ISSTANDING = 1
      Delay 1000
      exit
    end
  else
    if $isstanding=1//check if istanding=1
      Call hastarget
      if $havetarget=1
        SetConst isstanding=1
        keydown $restkey 500
        delay 900
        keydown $facekey 500
        while $havetarget=1
          keydown $specialattackkey 500
          delay 1000
          keydown $specialattackkey2 500
          delay 1500
          Call hastarget
        End//end of loop
      else
        keydown $restkey 500
        delay 800
        keydown $sprintkey 500
        SetConst ISSTANDING = 0
      end
    else
      Call hastarget
      if $havetarget=1
        SetConst isstanding=1
        keydown $restkey 500
        delay 900
        keydown $facekey 500
        while $havetarget=1
          keydown $specialattackkey 500
          delay 1000
          keydown $specialattackkey2 500
          delay 1500
          Call hastarget
        End//end of loop
      else
        keydown $sprintkey 500
        delay 800
      end
    End // end of ifstanding check
  End // end of full hp check
  delay 2000 // small delay, then recheck if full hp yet
  Call checkHP
End // end of checkHP
Procedure Pull
  call removepopup
  Call hastarget
  While $havetarget=0
    keydown $tabkey 500
    delay 2000
    call removepopup
    Call hastarget
  End //end of while havetarget
  Delay 900 //small delay after target is gotten
End // end of pull
Procedure Fight
  call removepopup
  Call checkengaged
  Call hastarget
  if $isengaged=1 //if#1
    if $havetarget=1
      Call Attack //calls attack funtion
    else
      keydown $attackkey 500 // to disengage
      exit
    End
  else //not engaged else#1
    if $havetarget=1 //if#2
      Call Attack
    else //else#2
      exit
    End //end of if#2
  End //end of if#1
End //end of fight procedure
Procedure Attack
  call removepopup
  keydown $facekey 500
  Delay 500
  Call checkmobhp
  if $mobhp=1
    keydown $pullkey 500
    delay 800
    keydown $pullkey 500
    delay 800
    keydown $pullkey 500
    delay 4200
    keydown $switch 500
    delay 2000
  end
  Loop 2
    keydown $specialattackkey 500
    delay 600
    keydown $specialattackkey2 500
    delay 1000
  End//end of loop
  Call Fight
End //end of attack procedure
